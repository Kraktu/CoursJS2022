Remplacement d'un son existant
...

Header :
...

Je vais dans ce fichier montrer la démarche pour remplacer un son existant dans notre projet.
*A noter qu'il est possible de faire d'autres techniques, pour rajouter un son non existant, mais ça demande une vision claire du SoundDesigner de l'utilisation de Unity, sinon ça risque d'être beaucoup de travail de développement pour rien, cela dit, on peut en discuter, mais sinon, pour le moment, impossible de rajouter un nouveau son (une nouvelle fonctionnalité de son je parle) sans l'aide d'un développeur !*

Body
...

Prérequis
...

  1. la nouvelle version du son à remplacer (ici je vais faire semblant de remplacer l'intro)

  1. GitHub for Desktop ouvert sur le projet symbiose et avoir pull

  1. Unity avec le projet ouvert dans les sons et sur la scène MainMenu !

Marche à suivre
...

  1. Dans les sons, aller dans le bon répertoire en fonction du type de son, le but sera de retrouver le son qu'on essaye de remplacer pour le supprimer après avoir tout réussi
  2. Glisser le nouveau son a côté de l'ancien
  3. Aller dans la hiérarchie, cliquer sur le SoundManager
  4. Sur la droite, dans l'inspecteur, descendre à l'endroit du Dictionnaire de SoundEffects
  5. Déplier la donnée qui nous intéresse à changer (dans notre cas MusicIntro)
  6. Changer le 2ème champs, celui contenant le son (le premier contient le nom, si on change ça le code plante !!)
  7. Tester le jeu pour aller écouter le nouveau son

Si on est content du résultat
...

  1. Supprimer l'ancien son équivalent du nouveau dans les assets
  2. Sauvegarder Unity
  3. Push sur Github (au début, il vaut mieux push trop souvent que trop peu, si tu as l'habitude avec Unity, alors tu peux faire plusieurs sons avant !)

Si on est pas content du résultat
...

  1. Refaire l'étape 3,4,5,6 avec l'ancien son qu'on a pas supprimé
  2. Supprimer le nouveau son des Assets
  3. Normalement, il n'y a aucun changement survenu sur Github ! (Changes)
  4. Dans le cas contraire du 10, me trouver, surtout avant de push quoi que ce soit de nouveau sur Github !!!!
